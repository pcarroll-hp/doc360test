<div class="sect1">
<h2 id="_product_terminology"><a class="anchor" href="#_product_terminology"></a>Product terminology</h2>
<div class="sectionbody">
<div class="paragraph">
<p>The HP Wolf Pro Security solution consists of 2 primary components:</p>
</div>
<div class="ulist">
<ul>
<li>
<p>HP Wolf Security Controller is your HP-cloud-hosted “controller” for administrators to manage the endpoint “agents”</p>
</li>
<li>
<p>HP Wolf Security is an “agent” that consists of several software features that are installed on individual end-user computers.</p>
<div class="ulist">
<ul>
<li>
<p>HP Wolf Pro Security Protection features</p>
</li>
<li>
<p>HP Wolf Security “Desktop Console” to review agent status or enable/disable features on a local device.</p>
</li>
<li>
<p>HP Sure Click Pro Secure Browser, a browser that uses Threat Containment features to open pages isolation. Additional browser extensions and an Outlook plug-in are also automatically installed</p>
</li>
</ul>
</div>
</li>
</ul>
</div>

<h2 id="_installing_the_endpoint_agent"><a class="anchor" href="#_installing_the_endpoint_agent"></a>Installing the endpoint agent</h2>
<div class="sectionbody">
<div class="paragraph">
<p>The endpoint agent installer is available for download as soon as your account is set up and before the controller instance has been created.
However, this installer needs to download certain product information and packages from the controller, so HP recommends that you run the installer only after the controller has been fully created.</p>
</div>
<div class="paragraph">
<p>The endpoint agent installer can be found on the controller <strong>Licenses</strong> page after you have logged in. If you already download the installer in prior steps (before the controller instance was created), you do not need to download it again. This installer, named <strong><em>HPSecurityUpdateService - [your tenant&#8217;s name will be here].msi</em></strong>, is only around 2MB in size and will quickly install on the computer. The installer will perform a series of checks and begin to download and install the agent to your computer shortly after you run it.</p>
</div>

<li>
<p>You will need to enter administrative credentials if you are currently a limited user of the computer.</p>
</li>
<li>

<div class="sect4">
<h5 id="_understanding_credential_protection_alerts"><a class="anchor" href="#_understanding_credential_protection_alerts"></a>Understanding Credential Protection alerts</h5>
<div class="paragraph">
<p>You can enable Credential Protection on devices using device group configuration.
When Credential Protection is enabled on a device, the HP Sure Click browser extension is installed on the available supported browsers.
Credential Protection is supported on the latest versions of Google Chrome, Mozilla Firefox and Microsoft Edge, and the HP Sure Click Pro Secure Browser.</p>
</div>
<div class="paragraph">
<p>When a user visits a website using a supported browser, Credential Protection checks the site&#8217;s risk classification:</p>
</div>
<div class="ulist">
<ul>
<li>
<p>If the website is not trusted and a user attempts to enter their credentials, a red warning overlay is displayed to inform them that the site is a known phishing site.
The user can choose to browse the site with inputs disabled but cannot log in.
Untrusted websites include:</p>
<div class="ulist">
<ul>
<li>
<p>Domains classified as "Blocked" in the portal.</p>
</li>
<li>
<p>Sites recognized by HP ThreatCloud as malicious.</p>
</li>
<li>
<p>Sites that the user has previously visited and marked as "Untrusted" in the browser extension.</p>
</li>
</ul>
</div>
</li>
<li>
<p>If the website is unknown (that is, it is not explicitly trusted or untrusted) and a user attempts to enter their credentials, a gray overlay is displayed warning the user that the site may be malicious.
The user can choose to either continue with login fields disabled, or mark the site as “trusted” in the Credential Protection browser extension.
The latter enables them to log in, and prevents the same warning from being displayed in future.
If the site is subsequently added to the portal block list or classified by HP ThreatCloud as malicious, this will override the user&#8217;s browser extension classification.
Unknown websites include:</p>
<div class="ulist">
<ul>
<li>
<p>Domains that are not currently classified in the portal or by HP ThreatCloud.</p>
</li>
<li>
<p>Sites that the user has not previously visited or which the user chose not to trust but has not explicitly blocked,</p>
</li>
</ul>
</div>
</li>
<li>
<p>If the website is trusted, the user can enter their credentials and log in as normal. Trusted websites include:</p>
<div class="ulist">
<ul>
<li>
<p>Domains classified as “Allowed” in the portal.</p>
</li>
<li>
<p>Sites included in the “Trusted websites” device group setting.</p>
</li>
<li>
<p>Sites that the user has previously visited and marked as “Trusted” in the browser extension.</p>
</li>
<li>
<p>Sites for which the browser has auto-filled the user’s credentials (indicating that the user has previously logged in).</p>
</li>
</ul>
</div>
</li>
</ul>
</div>
<div class="paragraph">
<p>When a user visits an unknown or untrusted site and attempts to enter credentials, a Credential Protection alert is raised.
You can review these alerts to see the sites your users are visiting that may pose a phishing risk, decide whether to allow or prevent users from logging into them in future, and advise users to change their passwords if necessary.</p>
</div>
<div class="paragraph">
<p><strong>Note:</strong> If the <strong>Permit users to disable HP Wolf Security features</strong> option has been enabled in Device Group configuration, end users can disable Credential Protection on their device by turning off the browser extension or disabling Sure Click Threat Containment from the Desktop Console or system tray.
You can check the Sure Click status from the <strong>Device Management Dashboard</strong>.
If the browser extension is turned off, a management action is raised for the device.</p>
</div>
</div>
<div class="sect4">
<h5 id="_triaging_credential_protection_alerts"><a class="anchor" href="#_triaging_credential_protection_alerts"></a>Triaging Credential Protection alerts</h5>
<div class="paragraph">
<p>A Credential Protection alert is raised if a user attempts to enter credentials on an unknown or untrusted website from a supported browser.
New alerts are classified automatically by HP ThreatCloud using curated threat data from a range of sources.
With the automatic classification as a starting point, you can manually classify alerts to determine whether sites should be trusted or not.</p>
</div>
<div class="paragraph">
<p>Alerts can be classified as follows:</p>
</div>
<div class="ulist">
<ul>
<li>
<p><strong>True Positive (TP):</strong> The website is malicious and poses a phishing risk. If the threat response is not “Blocked” or “Blocked by Blocking rule”, you should advise the user in question to change their password, as there is a high chance that their credentials have been harvested.
You may also want to add the site to the Domain Classification block list so that users cannot enter credentials on this site in future.</p>
</li>
<li>
<p><strong>False Positive (FP):</strong> The website is not malicious and does not pose a phishing risk.
You may also want to add the site to the Domain Classification allow list so that users can log in to this site normally in future.
This is useful for licensed SaaS products (such as accounting or HR systems) and sites which your employees regularly access for work as it avoids disruption to users' workflows and minimizes unnecessary alerts in the portal.</p>
</li>
<li>
<p><strong>Unknown:</strong> The website is neither trusted nor untrusted. This classification can be used to identify alerts that need to be reviewed further.</p>
</li>
</ul>
</div>
</div>
<div class="sect4">
<h5 id="_viewing_alert_details"><a class="anchor" href="#_viewing_alert_details"></a>Viewing alert details</h5>
<div class="paragraph">
<p>To view more information about an alert, click the alert to open the details page for the alert.
From here you can:</p>
</div>
<div class="ulist">
<ul>
<li>
<p>Apply a label to the alert to assist with triaging.</p>
</li>
<li>
<p>Assign the alert to a particular user for further review.</p>
</li>
<li>
<p>Classify the domain as either allowed, blocked or untriaged.</p>
</li>
<li>
<p>Classify the alert (or revert to the automatic classification).</p>
</li>
<li>
<p>Archive the alert so that it not longer appears in the alerts list by default.</p>
</li>
<li>
<p>Delete the alert to remove it from the portal.</p>
</li>
</ul>
</div>
<div class="paragraph">
<p><strong>Note:</strong> These actions can also be performed from the Alerts list.</p>
</div>
</div>
</div>
<div class="sect3">
<h4 id="_domain_classification"><a class="anchor" href="#_domain_classification"></a>Domain Classification</h4>
<div class="paragraph">
<p>The <strong>Domain Classification</strong> page allows you to control whether websites are blocked or allowed for the purposes of Credential Protection. This enables you to override sites which may be miscategorized or which you want to allow, such as an internal portal login.
For example:</p>
</div>
<div class="imageblock">
<div class="content">
<img src="./images/DomainClassification.jpg" alt="DomainClassification">
</div>
</div>
<div class="ulist">
<ul>
<li>
<p>If a domain is <em>blocked</em>, users can still visit the site but they cannot enter login credentials.
If a user attempts to enter credentials, a red warning overlay is displayed to inform them that the site is on the block list and an alert is raised.</p>
</li>
<li>
<p>If a domain is <em>allowed</em>, users can visit the site and enter login credentials as normal.
This is useful for internally hosted sites and SaaS products that your organization has licensed (such as accounting or HR systems).</p>
</li>
</ul>
</div>
<div class="paragraph">
<p>If a domain is classified as blocked or allowed, this overrides the classification provided by HP Threat Cloud and any local classification that a user has added to the Credential Protection browser extension.</p>
</div>
<div class="paragraph">
<p>To classify a domain:</p>
</div>
<div class="olist arabic">
<ol class="arabic">
<li>
<p>At the top of the page click <strong>Add Classification</strong>.</p>
</li>
<li>
<p>For <strong>Domain</strong>, enter the URL that you want to block or allow.</p>
</li>
<li>
<p>For <strong>Classification</strong>, select "Blocked" or "Allowed" as appropriate.
If you select "Untriaged", a classification is not created.</p>
</li>
<li>
<p>Click <strong>Confirm</strong>.
The domain is added to the list and the classification is applied immediately to devices on which Credential Protection is enabled.</p>
</li>
</ol>
</div>
<div class="paragraph">
<p>To change the classification of a domain, use the checkbox to select the domain and click the <strong>Delete</strong> icon.
Then add a new classification for the domain.</p>
</div>
<div class="ulist">
<ul>
<li>
<p><strong>Note:</strong> Websites included in the Trusted websites list of the Device Group configuration are also treated as "Allowed" for the purposes of Credential Protection.
If you want to allow users to log in to a site but do not want to confer the other privileges associated with the "Trusted websites" list (for example, trusting files downloaded from the site and disabling link protection), add the site to the Domain Classification "Allowed" list instead.</p>
</li>
</ul>
</div>
</div>
</div>
<div class="sect2">
<h3 id="_events"><a class="anchor" href="#_events"></a>Events</h3>
<div class="paragraph">
<p>Events are helpful for the device administrator or security specialist responsible for keeping track of metrics related to the health of the agent fleet or current deployment. It is also useful for specific questions such as “When was the last time a computer initialized?” or “How many devices have trusted a file located at C:\Windows\temp?”</p>
</div>
<div class="imageblock">
<div class="content">
<img src="./images/Events.jpg" alt="Events">
</div>
</div>
<div class="sect3">
<h4 id="_events_2"><a class="anchor" href="#_events_2"></a>Events</h4>
<div class="paragraph">
<p>The <strong>Events</strong> page provides a record of all the events that have occurred on end-user devices, categorized by severity. This can include:</p>
</div>
<div class="ulist">
<ul>
<li>
<p>User actions, such as disabling a Wolf Pro Security feature, trusting a file or website, or opening a quarantined file.</p>
</li>
<li>
<p>The outcome of remote commands issued from the portal, such as reinitializing or restarting Sure Click isolation.</p>
</li>
<li>
<p>The progress of automatic software updates.</p>
</li>
</ul>
</div>
<div class="paragraph">
<p>If an error or warning event requires further input, a management action is also raised.
Management actions are listed on the <strong>Device Dashboard</strong> and the <strong>Devices page</strong>.
Event data can be useful when troubleshooting issues or investigating the chain of events that led to a failure on a particular device.
Event data can also be useful when taking remedial action against a threat.
For example, you can use the Events list to identify all devices on which a particular file has been trusted.</p>
</div>
<div class="paragraph">
<p><strong>Note:</strong> The event timestamp reflects the time the event occurred converted to the portal&#8217;s default time zone. To view or change the default time zone, click your username and select Preferences.</p>
</div>
<div class="paragraph">
<p>You can configure and export the information displayed in the Events list as follows:</p>
</div>
<div class="ulist">
<ul>
<li>
<p>To filter the list by one or more parameters, click <strong>Add Filter</strong> and select the options you want to use.
The filter options are added to the taskbar.
Expand each filter option and select or enter the filter criteria and then click <strong>Apply</strong>.
To display items that do not match the filter criteria, expand the filter option and select <strong>Exclude from query</strong>.</p>
</li>
<li>
<p>To add, remove and change the order of columns, click <strong>Columns</strong> and drag items between the lists.</p>
</li>
<li>
<p>To save the filters, column order and sort order so that you can apply them easily in future, click <strong>Saved Views</strong> and select <strong>Save as</strong>.
Enter a name and description for the view and specify whether you want to make it the default view for all users of the portal, and then click <strong>Create saved view</strong>.
To load a saved view, click <strong>Saved views</strong> and select one from the list.</p>
</li>
<li>
<p>To copy the data currently displayed to the clipboard as tab-separated values, click the <strong>Copy</strong> icon.
To change the number of rows that are included, click <strong>Show n entries</strong> at the bottom of the list.</p>
</li>
<li>
<p>To download all rows of data to a .csv file, click the <strong>Export</strong> icon.
Only the currently displayed columns are included in the export.</p>
</li>
</ul>
</div>
</div>
<div class="sect3">
<h4 id="_email_destinations"><a class="anchor" href="#_email_destinations"></a>Email destinations</h4>
<div class="paragraph">
<p>The <strong>Email Destinations</strong> page allows you to configure email alerts to notify you when events are recorded.
You can configure alerts based on the source and severity of events.
For example, you may want to send email alerts to specific individuals when errors from Sure Click Threat and Sure Sense are raised while sending installer and initialization errors and warnings to a group inbox.</p>
</div>
<div class="paragraph">
<p>To add an email destination:</p>
</div>
<div class="olist arabic">
<ol class="arabic">
<li>
<p>At the top of the <strong>Email Destinations</strong> page click <strong>Add Email Destination</strong>.</p>
</li>
<li>
<p>Enter a name for the destination to help you identify it in future.</p>
</li>
<li>
<p>Select Enable to enable the email alert.
If this option is left unchecked, the alert is not active and emails will not be sent to the specified recipients when events are recorded.</p>
</li>
<li>
<p>Enter an email address and click <strong>Add Email</strong>.
To add additional email addresses, repeat this step.</p>
</li>
<li>
<p>Click <strong>Send Test Email</strong> to check that the server can send to the addresses specified.</p>
</li>
<li>
<p>For each event source, select the severity of alert that you want to send to the recipients:</p>
<div class="ulist">
<ul>
<li>
<p>Select "Error" to send emails only when an error is recorded.</p>
</li>
<li>
<p>Select "Warning" to send emails each time an error or warning is recorded.</p>
</li>
<li>
<p>Select "Ignore" if you do not want to send emails for events from this source.</p>
</li>
</ul>
</div>
</li>
<li>
<p>Click <strong>Save</strong>. The destination is added to the list.</p>
</li>
<li>
<p>(Optional) Click an item in the list to edit the alert settings or delete the alert.</p>
</li>
</ol>
</div>
</div>
</div>
<div class="sect2">
<h3 id="_accounts"><a class="anchor" href="#_accounts"></a>Accounts</h3>
<div class="paragraph">
<p>The <strong>Accounts</strong> page enables you to grant colleagues access to the HP Wolf Partner Portal.</p>
</div>
<div class="paragraph">
<p>When you initially setup your controller (HP or a Partner may have initiated this on your behalf) you had the option to add users. You can still add users at any time if your access has “Customer Administrator” power.</p>
</div>
<div class="olist arabic">
<ol class="arabic">
<li>
<p>Navigate to the <strong>Accounts</strong> page.</p>
</li>
<li>
<p>Select <strong>Add Account</strong>.</p>
</li>
<li>
<p>Provide an email address for the new user and the level of access you would like to grant.</p>
</li>
</ol>
</div>
<div class="paragraph">
<p>There are 2 levels of access you can assign:</p>
</div>
<div class="ulist">
<ul>
<li>
<p><strong>Customer Administrator:</strong> Administrator can make changes in the Controller.</p>
</li>
<li>
<p><strong>Customer Read Only:</strong> Can only view the Controller settings and reports.</p>
</li>
</ul>
