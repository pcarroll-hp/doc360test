<div class="paragraph">
<p>This management action is reported when Sure Run fails to activate due to an unexpected error on the endpoint.</p>
</div>
<div class="paragraph">
<p>The device event log on the Wolf Security Controller contains specific information about the error. Review the log to help identify the underlying cause of the error. You can also verify the current Sure Run status on the device details page in Wolf Security Controller.</p>
</div>
<div class="sect1">
<h2 id="_remediation_options">Remediation options</h2>
<div class="sectionbody">
<div class="paragraph">
<p>If the reported status does not match the expected policy configuration, try the following steps:</p>
</div>
<div class="ulist">
<ul>
<li>
<p>Reboot the device. In some cases, Sure Run requires a reboot to transition into the correct operational state.</p>
</li>
<li>
<p>Ensure the device BIOS is up to date. An outdated BIOS can prevent Sure Run from functioning correctly.</p>
</li>
<li>
<p>If the issue persists, remove and then reapply the Sure Run policy to the affected device group.</p>
</li>
</ul>
</div>
<div class="paragraph">
<p>If the error continues after completing these steps, it needs further investigation.
Collect a log bundle, the device event logs, and contact HP Wolf Support.</p>
</div>
